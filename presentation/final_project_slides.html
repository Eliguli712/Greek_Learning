Semantic DAG Compiler for Ancient Greek

Why Analyze Ancient Greek Semantics?

  • Classical texts are often studied through translations, commentaries, and lexical notes
  • Researchers may want explicit structure: who did what, to what, and with which modifiers
  • Ancient Greek is morphologically rich, so surface word forms hide useful information
  • A graph representation can expose entities, events, semantic roles, and dependencies

What We Want To Produce

Input

  • Ancient Greek tokens
  • Lemmas, POS tags, and morphological features
  • Dependency links from real UD treebanks

Output

  • One graph node per linguistic token
  • Semantic node types: event, entity, property, connector
  • Semantic edges: AGENT, THEME, MODIFIER, COMPLEMENT, COORD
  • A structurally valid directed acyclic graph
Goal: make predicate-argument structure inspectable as a graph instead of a flat list of tags.

Previous Work

  • Universal Dependencies: cross-linguistically consistent dependency annotation for POS, morphology, and syntax (Nivre et al., 2016)
  • Ancient Greek treebanks: Perseus-style and PROIEL treebanks provide real Classical/Koine Greek annotations with lemmas, morphology, and dependencies (Bamman & Crane, 2011; Haug & Jøhndal, 2008)
  • Semantic role labeling: PropBank-style work adds predicate-argument role annotations on top of syntactic structure (Palmer et al., 2005)
  • Semantic graph representations: AMR represents sentence meaning as graph structure over events, entities, and relations (Banarescu et al., 2013)
  • Our approach: compile Ancient Greek linguistic annotations into an inspectable semantic DAG with explicit role-labeled edges

How The DAG Builder Works

  • Nodes: each UD token becomes one graph node with token, lemma, POS, morphology, semantic type, and confidence
  • Edges: mapped dependencies become directed semantic edges such as AGENT, THEME, MODIFIER, COMPLEMENT, and COORD
  • Indexing: token positions are converted into stable node IDs such as n0, n1, and n2
  • Cleanup: duplicate edges, self-loops, and out-of-range links are removed before validation
  • Validation: NetworkX checks acyclicity, allowed edge labels, valid endpoints, and required predicate arguments

Conclusion

  • Main result: real Ancient Greek UD annotations can be compiled into a semantic DAG with high content-relation coverage
  • Representation: lemma, POS, morphology, and dependency structure are preserved in one graph object
  • Semantic abstraction: syntactic dependencies are reorganized as AGENT, THEME, MODIFIER, COMPLEMENT, and COORD edges
  • Evaluation: 99.49% content-relation F1 and 99.98% predicate-argument F1 on real test data
  • Use case: the output is inspectable, visualizable, and suitable for downstream graph comparison or analysis
