Ancient Greek raw analyzer update

Local UD-derived lexicon dramatically improves real-sentence analysis

The analyzer now reads local Universal Dependencies data and uses it as a corpus lexicon for lemma, UPOS, morphology, semantic typing, and relation prediction.

0.9409 semantic type accuracy after lexicon integration
0.2992 relation F1 after relation-rule update

What Changed

The core change is a local lexicon built from data/real_eval/*.conllu. Each Greek surface form is mapped to UD lemma, UPOS, and morphology, then reused by the raw analyzer instead of relying mostly on fallback guesses.

UD corpora data/real_eval/*.conllu provides Greek forms, lemmas, UPOS tags, and morphological features.
→
Local lexicon src/corpus_lexicon.py normalizes and indexes the observed forms for lookup.
→
Analyzer stages Lexeme normalization, semantic typing, and relation prediction consume the corpus-backed features.

Lexeme Normalization

The lexeme layer can now resolve observed Greek surface forms to UD lemmas and morphological features before falling back to less precise similarity or unknown-token behavior.

Semantic Defaults

Additional UPOS-to-semantic defaults improve proxy semantic labels for nouns, verbs, modifiers, function words, and other coarse syntactic classes.

Relation Prediction

Relation rules now use UD Case information and skip function words, reducing noisy edges before semantic graph scoring.

Added Source File

The corpus-backed lookup logic lives in a dedicated module so the analyzer can load UD-derived lexical hints without mixing corpus parsing into the pipeline stages.

src/corpus_lexicon.py

Snapshot note

The command is intentionally identical for before and after. The measured change comes from the code snapshot: pre-lexicon analyzer versus the analyzer with src/corpus_lexicon.py, lexeme lookup integration, expanded UPOS defaults, and case-aware relation rules.

Interpretation

Why token-level metrics jumped

Lemma, UPOS, and semantic type scoring benefit directly from the local UD-derived lexicon. Once a surface form is found in the lexicon, the analyzer receives high-quality structured features instead of reconstructing them from sparse rules.

Why relation F1 is still lower

Relation prediction is a harder graph task. Case-aware rules and function-word filtering reduce noise, but edge labels still depend on syntactic attachment, semantic role heuristics, and clause structure.

The strongest result is that corpus-backed lexical information makes the raw analyzer viable on real Ancient Greek UD sentences. The remaining bottleneck is semantic relation inference, where the system needs stricter attachment logic and broader role-specific evaluation.

Evaluation caveat

Because the lexicon is derived from local UD data, the current result may benefit from overlap between lookup data and evaluation data. A later train/test split should build the lexicon from one partition and evaluate on held-out sentences for a stricter estimate.
